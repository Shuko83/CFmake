Pour développer :
Definir 3 variables d'environnement
SW_BIN_D pointe sur mscv2013/bin/debug ou mscv2008/bin/debug
SW_BIN_R pointe sur mscv2013/bin/debug ou mscv2008/bin/release
SW_INCLUDE pointe sur src

Ajouter SW_BIN_D et _R au PATH

Pour les includes, dans un .pro faire comme pour boost; en utilisant les variables d'environnement.
A condition de savoir dans quel module est quel fonctionnalité de Sw (cf #7815 revoir documentation)

Exemple, pour créer un nouveau composant ExampleComponent basé sur SwAssistedComponent qui est dans SwFoundation, en mode debug

INCLUDEPATH += $$(SW_INCLUDE)
LIBS += -L"$$(SW_BIN_D)"
LIBS += -lSwFoundationd

Puis dans ExampleComponent.h faire #include "SwFoundation/SwAssistedComponent.h"

Pour garder compatibilité avec le code actuel, ie qui ecrit plutot #include "SwAssistedComponent.h" on fera

INCLUDEPATH += $$(SW_INCLUDE)\SwFoundation
LIBS += -L"$$(SW_BIN_D)"
LIBS += -lSwFoundationd
