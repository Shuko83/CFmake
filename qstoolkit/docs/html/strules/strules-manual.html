ST Specific Rules

Overview

ST specific rules override QS features by defining additional rules common to development projects of ST division. Two set of rules, implemented in qmake project include files (.pri), are available in the $${QSMAKEPATH}/rules directory:

  • standard project rules (standard_project.pri file), which define variables and compiler rules specific to any ST development project.
  • LDTools project rules (ldtools_project.pri file), based on standard project rules, which define additional features specific to project files generated with LDTools.

Prerequisites

  • QS qmake features must be installed. See features installation section for more details.
  • Any project source tree must contain only one qmake project file (.pro).
  • qmake project file has to include the desired ST specific rules file, at least after target-specific variables definition. Prefers use QSMAKEPATH variable to retrieve the rules file location.

For example, for a project including standard project rules:

# Target Definition

TARGET = MyLib
TEMPLATE = lib
CONFIG += shared

INSTALLDIR = ../../Delivery/AppDir

# Sources Definition

HEADERS = ...
SOURCES = ...

# ST Rules Include

include( $${QSMAKEPATH}/rules/standard_project.pri )
